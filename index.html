2021-03-21

缓冲区buffer (缓存cache)

* 内存的一部分，用于让低速的输入输出设备与高速的程序协调工作，降低设备的读写次数，提高工作效率

* 应用程序不能直接读写硬件，告诉操作系统，让内核kernel调用驱动程序，才能操作硬件

全缓冲

1k 4m等被填满后才进行输入或输出操作

行缓冲

  • 标准输入设备(键盘),标准输出设备(显示器)

遇换行符(\n)结束,按下回车产生换行符

不带缓冲

perror()，getche(), getch()等

清空(刷新)

fflush(stdout)(LLVM/Clang不支持)

  • 非标准，各大编译不通用

while((c=getchar())&&!=\n&&c!=EOF)

  • 优:通用，高效
    缺:需要额外定义一个字符变量

%[*][宽度][长度]

  • []表示任选项
    • 读入后不赋予变量

scanf(“%*[^\n]”); 清空buf中\n前的字符

scanf(“%*”); 清空换行符符

  • 通用，高效，无需要另外定义一个字符

2021-03-19

array

数组与字符串

一维数组

type arr[元素个数]

初始值

赋值元素个数小于总元素个数时

  • short int long初始化为0

  • float double初始化为0.0

  • char初始化为’\n’

不进行赋初值时,编译器将局部数据(变量、数组、结构体、共用体)赋值为随机的垃圾值

二维数组

一个二维数组

type arr[数组行数][数组列数]

两种存储方式
* 按行:一行一行存储

二维数组的第一行元素: arr[0][0]、arr[0][1]、arr[0][2]、arr[0][n]

* 按列:一列一列存储

二维数组的第一列元素: arr[0][0]、arr[1][0]、arr[2][0]、arr[n][0]

拆解成多个一维数组

type arr[ 数组个数 ][ 数组元素个数 ]

第一个一维数组arr[0] 的元素有arr[0][0]、arr[0][1]、arr[0][2]、arr[0][n]

字符串

定义与初始化

  • 开发常用char str[n] = { “hello” }

  • 开发常用 char str[]=”hello”;

  • 所在内存的字符串有读写权限

  • char *str = “abc”，所在内存的字符串只有读权限

  • 未进行初始化时，赋值元素个数小于字符串长度时,剩余元素是随机的垃圾值,字符串的长度会改变,直到遇到\0,有影响其他数据的危险

专业初始化

  • 赋值为0

  • ASCII码第0个字符’\0’,(空字符NUL),编码值为0(0==’\0’)
    只作为字符串结束标志,并且占字符串的一个字符位置，无显示无控制功能,只是一个标志

  • 字符串的长度不包括空字符\0,空字符只是占了一个数组元素的位置,不在字符串里

  • 口 表情无法显示的特殊字符
